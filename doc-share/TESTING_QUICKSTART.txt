pseudoswapper — Manual Testing Quick Start
==========================================

ENVIRONMENT SETUP (once per terminal session)
----------------------------------------------
cd ~/CodingProjects/pseudoswapper
source .venv/bin/activate

Your prompt will show (.venv). After activation, pseudoswapper is a real shell
command — no python prefix needed. To confirm it's working:

    pseudoswapper --help


CONFIG FILE (one-time setup)
-----------------------------
If you haven't already created your config:

    cp pseudoswapper_config.example.yaml ~/.pseudoswapper_config.yaml

Then edit it to add your company terms and known employees:

    pseudoswapper config --edit      # opens in $EDITOR
    pseudoswapper config --show      # print the active config


DOCUMENT MODE
-------------
Redact the included sample fixture:

    pseudoswapper document tests/fixtures/sample_document.txt

Output written to: tests/fixtures/sample_document.redacted.txt

Inspect the redacted output:

    cat tests/fixtures/sample_document.redacted.txt

Restore it (simulating AI returning the file):

    pseudoswapper restore tests/fixtures/sample_document.redacted.txt

Output written to: tests/fixtures/sample_document.redacted.restored.txt

Confirm original values are back:

    cat tests/fixtures/sample_document.redacted.restored.txt


STRUCTURED MODE (CSV)
---------------------
Redact the sample structured fixture:

    pseudoswapper structured tests/fixtures/sample_structured.csv --anchor employee_id

Output written to: tests/fixtures/sample_structured.redacted.csv

    cat tests/fixtures/sample_structured.redacted.csv

Restore:

    pseudoswapper restore tests/fixtures/sample_structured.redacted.csv
    cat tests/fixtures/sample_structured.redacted.restored.csv


STRUCTURED MODE (JSON and XLSX)
--------------------------------
    pseudoswapper structured tests/fixtures/sample_structured.json --anchor employee_id
    pseudoswapper structured tests/fixtures/sample_structured.xlsx --anchor employee_id


SESSION INSPECTION
------------------
After any redact command, a .pseudoswapper_session pointer file is created in
the current directory. You can inspect it:

    cat .pseudoswapper_session       # shows path to the private temp dir

The pointer and temp dir are deleted automatically after a successful restore.
To abandon a session manually:

    pseudoswapper clear-session


IMPORTANT: RESTORE FROM THE SAME DIRECTORY
-------------------------------------------
pseudoswapper restore looks for .pseudoswapper_session in the current working
directory. Always run restore from the same directory where you ran the redact
command, or it will not find the session.


OPENING A NEW TERMINAL TAB
---------------------------
You need to re-activate the venv each time:

    cd ~/CodingProjects/pseudoswapper
    source .venv/bin/activate

The config file (~/.pseudoswapper_config.yaml) and installed command persist
across sessions — only the venv activation needs to be repeated.


OUTPUT FILE NAMING
------------------
Redact:   name.txt        → name.redacted.txt
          data.csv        → data.redacted.csv
          data.xlsx       → data.redacted.xlsx

Restore:  name.redacted.txt → name.redacted.restored.txt

Output is always written alongside the input file in the same directory.
